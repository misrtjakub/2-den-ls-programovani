Složené přiřazení

Zkrácený zápis pro operaci a přiřazení zároveň

Co je složené přiřazení?

  • x += 5; je zkratka pro x = x + 5;
  • Ušetří místo a je přehlednější
  • Funguje s různými operacemi: +, -, *, /, %

Nejběžnější operátory

  • += přičte hodnotu
  • -= odečte hodnotu
  • *= vynásobí hodnotu
  • /= vydělí hodnotu
  • %= zbytek po dělení

Příklad: += a -=

int value = 10;

value += 5;  // value = 15
value -= 3;  // value = 12

Console.WriteLine(value);

Příklad: *= a /=

int number = 4;

number *= 2;  // number = 8
number /= 4;  // number = 2

Console.WriteLine(number);

Příklad: %= (zbytek po dělení)

int total = 10;

total %= 4;  // total = 2 (zbytek z 10 / 4)

Console.WriteLine(total);

Proč se to hodí?

  • Zápis je kratší a přehlednější
  • Zlepšuje čitelnost kódu

příklad

int sum = 0;

for (int i = 1; i <= 5; i++) {
    sum += i;  // přičítá čísla 1 až 5
}

Console.WriteLine("Součet: " + sum);  // 15

Binární soustava

  • Používá jen dvě číslice: 0 a 1
  • Každá pozice má hodnotu jako mocnina 2
  • Používá se uvnitř počítačů pro reprezentaci dat

Jak spočítat binární číslo?

Příklad: 1011

  • 1 × 2³ = 8
  • 0 × 2² = 0
  • 1 × 2¹ = 2
  • 1 × 2⁰ = 1
  • Výsledek: 8 + 0 + 2 + 1 = 11

Jak převést z desítkové do binární?

Příklad: 13

  • 13 ÷ 2 = 6 zbytek 1
  • 6 ÷ 2 = 3 zbytek 0
  • 3 ÷ 2 = 1 zbytek 1
  • 1 ÷ 2 = 0 zbytek 1
  • Výsledek: 1101 (čteme zespodu nahoru)

Binární operace

  • Operace na úrovni bitů
  • Používají se hlavně s celými čísly
  • Užitečné např. pro ukládání oprávnění nebo zapínání funkcí (např. zvuk, tmavý režim)
  • Používá se i v kryptografii nebo při optimalizaci dat (např. komprese, šifrování)

Základní binární operátory

  • & (bitové AND)
  • | (bitové OR)
  • ^ (bitové XOR)
  • ~ (bitová negace)
  • << (posun doleva)
  • >> (posun doprava)

& (bitové AND)

  • Porovná každý bit dvou čísel
  • Výsledek je 1 jen pokud jsou oba bity 1
  • Používá se pro zjištění, zda jsou určité bity nastavené

| (bitové OR)

  • Porovná každý bit dvou čísel
  • Výsledek je 1, pokud aspoň jeden bit je 1
  • Používá se pro zapnutí určitých bitů

^ (bitové XOR)

  • Porovná každý bit dvou čísel
  • Výsledek je 1, pokud se bity liší
  • Používá se např. v kryptografii

~ (bitová negace)

  • Převrátí každý bit čísla
  • Z jedničky udělá nulu a obráceně
  • Mění i znaménko čísla

<< (posun doleva)

  • Posune všechny bity doleva o daný počet míst
  • Výsledkem je vynásobení čísla mocninou 2
  • Rychlá alternativa k násobení

>> (posun doprava)

  • Posune všechny bity doprava o daný počet míst
  • Výsledkem je celočíselné dělení čísla mocninou 2
  • Rychlá alternativa k dělení

Příklad

int a = 5;   // 0101
int b = 3;   // 0011

int and = a & b;   // 0001 = 1
int or = a | b;    // 0111 = 7
int xor = a ^ b;   // 0110 = 6
int notA = ~a;     // 11111111 11111111 11111111 11111010 = -6
int shiftLeft = a << 1;  // 1010 = 10
int shiftRight = a >> 1; // 0010 = 2

ÚKOL

1. Vytvoř dvě proměnné: int a = 12; a int b = 5;

2. Pomocí binárních operátorů spočítej a vypiš:

  • a & b
  • a | b
  • a ^ b
  • a << 1
  • a >> 1

Parsing v C#

Jak převést string na číslo a nezabít program

Proč potřebujeme parsing?

  • Všechny vstupy z konzole jsou typu string
  • Ale my pracujeme s typy jako int, double, decimal
  • Musíme tedy převést (parsovat) řetězec na číselnou hodnotu

Metoda Parse()

int i = int.Parse("123");    // OK
int j = int.Parse("abc");    // Program spadne
  • Parse() je jednoduché
  • ALE způsobí pád programu, pokud vstup není validní

Obecný tvar metody Parse

int x = int.Parse("123");
double y = double.Parse("3.14");
decimal z = decimal.Parse("100.25");

Všechny metody mají tvar: typ.Parse(string)

Co když uživatel zadá neplatný vstup?

Program spadne

string input = "neco";
int number = int.Parse(input); // Výjimka: Formátová chyba

Řešením je použití TryParse()

Bezpečné parsování: TryParse()

string s = "42";
bool ok = int.TryParse(s, out int x);

if (ok) {
    Console.WriteLine("Zadal jsi číslo: " + x);
} else {
    Console.WriteLine("Neplatný vstup.");
}
  • Vrací true pokud převod proběhl úspěšně
  • Výsledek převodu je uložen do x

Obecný tvar TryParse

// Int:
bool b1 = int.TryParse(s, out int x);

// Decimal:
bool b2 = decimal.TryParse(s, out decimal d);

Výstupní proměnná x je naplněna hodnotou nebo nulou

Jak funguje out v TryParse

  • Proměnná je předána jako „prázdná schránka“
  • Metoda TryParse se do ní pokusí uložit výsledek
  • Pokud se převod podaří:
    • → proměnná obsahuje převedené číslo
    • → TryParse vrátí true
  • Pokud se převod nepodaří:
    • → proměnná zůstane na výchozí hodnotě (např. 0)
    • → TryParse vrátí false

Ukázka použití v podmínce

Console.Write("Zadej číslo: ");
string input = Console.ReadLine();

if (int.TryParse(input, out int number)) {
    Console.WriteLine("Díky! Zadals číslo: " + number);
} else {
    Console.WriteLine("Tohle nebylo platné číslo.");
}

TryParse = Robustní kód

  • Nezáleží, co uživatel zadá
  • Program nespadne
  • Na neplatný vstup můžeme reagovat
  • Uživatelsky přívětivější a odolnější aplikace

Pole v C#

Umožňují uložit více hodnot do jedné proměnné

Co je pole?

  • Více prvků stejného typu
  • Přístup přes index (od 0)
  • Velikost je pevná

Příklad: Dny v týdnu

string[] daysOfWeek = {
    "pondělí", "úterý", "středa",
    "čtvrtek", "pátek", "sobota", "neděle"
};

Console.WriteLine(daysOfWeek[0]); // pondělí
Console.WriteLine(daysOfWeek[6]); // neděle

Deklarace + pozdější inicializace

string[] months;
months = new string[12];

Pole má danou velikost a prázdné hodnoty

Počet prvků v poli

int[] daysInMonths = { 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31 };

Console.WriteLine(daysInMonths.Length); // 12
Console.WriteLine(daysInMonths[0]);     // 31
  • .Length říká, kolik hodnot pole obsahuje
  • V tomto případě: 12 měsíců = 12 čísel

Úkol

  • Vytvoř pole s názvy měsíců (12 prvků)
  • Vytvoř pole s počty dnů v měsících
  • Vypiš, kolik dnů má červenec a listopad pomocí pole

Co je to List?

  • Flexibilní seznam hodnot (podobný poli)
  • Nemusíme znát předem velikost
  • Potřebuje knihovnu: using System.Collections.Generic;

Vytvoření Listu

List<int> numbers = new List<int>();
List<int> numbers2 = new List<int> { 10, 20, 30 };

Přidávání a přístup

numbers.Add(40); // přidá 40 na konec seznamu
int prvni = numbers[0]; // přístup k prvnímu prvku
int kolik = numbers.Count; // počet prvků v seznamu

Mazání prvků

numbers.Remove(20);    // smaže první výskyt 20
numbers.RemoveAt(1);   // smaže prvek na indexu 1

ÚKOL

1. Vytvoř List s čísly: 100, 200, 300

2. Přidej číslo 400

3. Smaž 200

4. Vypiš počet prvků v listu

5. Vypiš první prvek

Co je cyklus?

  • Umožní opakovat stejný kód víckrát
  • Šetří čas i místo v programu
  • Hodí se třeba pro práci s polem nebo Listem

Cyklus for

for (int i = 0; i < 5; i++) {
    Console.WriteLine("Ahoj");
}

Vypíše „Ahoj“ pětkrát

Procházení pole pomocí for

string[] names = { "Anna", "Tom", "Eva" };

for (int i = 0; i < names.Length; i++) {
    Console.WriteLine(names[i]);
}

Vypíše všechna jména

Cyklus foreach

List<int> scores = new List<int> { 5, 3, 1 };

foreach (int score in scores) {
    Console.WriteLine(score);
}

Pro každý prvek v Listu udělá to stejné

Cyklus while

int x = 0;

while (x < 3) {
    Console.WriteLine("Zdar");
    x++;
}

Opakuje se, dokud x < 3

Úkol

Vytvoř pole se 3 zvířaty.

Pomocí cyklu for i foreach je všechny vypiš do konzole.

Git a GitHub

Nejlepší přítel vývojáře

Git

  • Nástroj na verzování
  • Umožňuje sledovat změny v kódu
  • Pomáhá dokumentovat vývoj
  • Umožňuje spolupráci více lidí na jednom projektu

GitHub

  • GitHub je (cloud-based) vývojářská platforma využívající nástroj Git
  • Přináší super flexibilní spolupráci
  • Umožňuje správu verzí, issues, pull requestů a týmovou práci
  • Nabízí další nástroje jako např. GitHub Actions pro automatizaci

Git - Terminologie

  • Git repo (repozitář)
  • Složka, ve které se inicializuje Git repozitář

Clone

  • Stáhnutí repozitáře k sobě na zařízení

Commit

  • „Aktualizace“ kódu
  • Zaznamenání změn v repozitáři

Push

  • Odeslání změn do vzdáleného repozitáře (např. GitHub)
  • Musíš mít práva k zápisu

Pull

  • Stáhnutí aktualizací z repozitáře
  • Používá se po klonování nebo při aktualizaci

Fork

  • Vytvoření vlastní kopie cizího repozitáře
  • Používá se při práci na Open Source projektech

Open-Source

  • Projekt s veřejným kódem
  • Licence povoluje používání, úpravy i sdílení
  • Příklad: Android je postavený na Linuxu

Branch (větev)

  • Alternativní verze kódu
  • Např. „main“, „dev“, „feature-login“
  • Každá větev může obsahovat jiné změny

Merge

  • Spojení změn z jedné větve do druhé
  • Často se spojuje „dev“ do „main“
  • Pozor na konflikty!
